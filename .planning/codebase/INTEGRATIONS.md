# Integrations

## Filesystem

**Primary data store.** All notes, comments, and source configuration live on the filesystem.

| Operation | Implementation |
|---|---|
| Note scanning | `os.walk()` over `NOTES_ROOT`, filtering by `.md` extension |
| Note reading | `pathlib.Path.read_text()` |
| Note writing | `pathlib.Path.write_text()` |
| Comment storage | Sidecar `.{stem}.comments.json` files via `json.dumps/loads` |
| Source config | `tool/sources.json` via `json.dumps/loads` |
| Directory creation | `pathlib.Path.mkdir()` |
| File renaming | `pathlib.Path.rename()` (also moves sidecar files) |
| File serving | `flask.send_from_directory()` for images and other assets |

Skip directories: `.git`, `.venv`, `__pycache__`, `node_modules`

## Git

**Version control for notes**, driven from the UI via subprocess calls.

| Operation | Command | Working Dir |
|---|---|---|
| Repo detection | `git rev-parse --git-dir` or check `.git` dir | `NOTES_ROOT` |
| Status check | `git status --porcelain .` | `NOTES_ROOT` |
| Stage all | `git add .` | `NOTES_ROOT` |
| Commit | `git commit -m "<auto-generated message>"` | `NOTES_ROOT` |
| Push | `git push` | `NOTES_ROOT` |
| Pull | `git pull` | `NOTES_ROOT` |

Key behaviors:
- Git operations target the **active notes source**, not the SlobNotes repo itself
- Commit message auto-generated: `"notes updated.\n\n- file1\n- file2"`
- Push failure after successful commit returns `{"ok": true, "pushError": "..."}` (non-fatal)
- Pull conflict detection: checks for `"CONFLICT"` in stdout or non-zero returncode

## CDN (Content Delivery)

| Resource | CDN | URL |
|---|---|---|
| marked.js | jsDelivr | `https://cdn.jsdelivr.net/npm/marked/marked.min.js` |
| Space Grotesk font | Google Fonts | `https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;600;700` |
| Space Mono font | Google Fonts | `https://fonts.googleapis.com/css2?family=Space+Mono:wght@400;700` |

No version pinning on any CDN resource. App requires internet for fonts and markdown rendering.

## Netlify

**Static site deployment** configured in `netlify.toml`.

| Setting | Value |
|---|---|
| Build command | `git clone $NOTES_REPO /tmp/notes && pip install cryptography && NOTES_ROOT=/tmp/notes python3 tool/scripts/build_static.py` |
| Publish directory | `tool/dist` |
| Python version | 3.12 |

Required environment variables (set in Netlify dashboard):
- `NOTES_REPO` — git URL for the notes repository
- `SITE_PASSWORD` — encryption password for AES-256-GCM
- `SITE_TITLE` — optional, defaults to "slob notes"

## Browser APIs

| API | Usage |
|---|---|
| `localStorage` | Persist palette preference (`notesViewerPalette` key) |
| `fetch` | All API communication (no XMLHttpRequest) |
| `Web Crypto API` | AES-GCM decryption + PBKDF2 key derivation (static build only) |

## No External Service Integrations

- No database (filesystem only)
- No authentication service
- No analytics
- No error tracking
- No WebSocket / real-time sync
