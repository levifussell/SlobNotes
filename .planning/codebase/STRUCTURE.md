# Structure

## Directory Layout

```
SlobNotes/
├── CLAUDE.md                          # Project docs (workspace config)
├── netlify.toml                       # Netlify deploy config
├── .gitignore                         # Ignores .venv, __pycache__, tool/dist/, notes/, sources.json
├── .venv/                             # Python virtualenv (gitignored)
└── tool/
    ├── sources.json                   # Multi-source config (gitignored, auto-created)
    ├── scripts/
    │   ├── run.sh                     # Launch script (activates venv, runs server)
    │   ├── install.sh                 # Creates venv, installs requirements
    │   └── build_static.py            # Encrypted static site builder (for Netlify)
    └── viewer/
        ├── requirements.txt           # Python deps (just "flask")
        ├── server.py                  # Flask backend — all API routes
        └── static/
            ├── index.html             # SPA HTML shell
            ├── app.js                 # All frontend logic (~1030 lines)
            ├── style.css              # Layout and component styles (~870 lines)
            └── theme.css              # CSS custom properties (colors, fonts, spacing)
```

## Key Entry Points

| Entry Point | Purpose |
|---|---|
| `tool/scripts/run.sh` | Primary launch — activates venv, runs `server.py` |
| `tool/viewer/server.py` | Can also run directly: `python3 tool/viewer/server.py` |
| `tool/scripts/build_static.py` | Static site build for deployment: `SITE_PASSWORD=x python3 tool/scripts/build_static.py` |
| `tool/scripts/install.sh` | First-time setup — creates `.venv`, installs Flask |

## File Responsibilities

### Backend (single file)

- **`server.py`** (~585 lines) — Everything: Flask app creation, note scanning, tag extraction, all 18 API routes, git operations, comment CRUD, source management, path validation, in-memory caching

### Frontend (3 files, no build step)

- **`app.js`** (~1030 lines) — All client-side logic: state management, API calls, DOM manipulation, tag filtering, markdown rendering, comments UI, palette cycling, source selector, keyboard shortcuts, resize handle, mobile sidebar, title editing
- **`style.css`** (~870 lines) — Full layout: app shell, toolbar, panels, tag pills, note list, editor, rendered markdown styles, comments, modals, scrollbar, mobile responsive
- **`theme.css`** (~47 lines) — CSS custom properties only: colors (Pico-8 palette), fonts (Space Grotesk + Space Mono via Google Fonts), spacing tokens

### Build/Deploy

- **`build_static.py`** (~710 lines) — Duplicates tag extraction logic from `server.py`, adds AES-256-GCM encryption, generates read-only versions of `index.html` and `app.js` with password gate and Web Crypto API decryption
- **`netlify.toml`** — Build command clones `$NOTES_REPO`, encrypts with `$SITE_PASSWORD`, publishes `tool/dist/`

## Generated/Runtime Paths

| Path | Generated By | Contents |
|---|---|---|
| `tool/dist/` | `build_static.py` | `data.enc`, `index.html`, `app.js`, `theme.css`, `style.css` |
| `tool/sources.json` | Server at runtime | `{sources: [{name, path}], active: "path"}` |
| `notes/` | Default notes dir | Only used if no source configured and `notes/` exists |
